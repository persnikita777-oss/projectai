# ProjectAI.ru — Документация проекта

## Концепция

**ProjectAI.ru** — AI-студия полного цикла для малого и среднего бизнеса в России.
AI-ассистент на сайте автоматически принимает заявки, собирает ТЗ через диалог, генерирует план работ и коммерческое предложение.

**Миссия:** Сделать AI-разработку доступной для бизнеса любого размера — от ИП до среднего бизнеса.

**УТП (уникальное торговое предложение):**
- AI-ассистент вместо формы заявки — клиент получает предварительную оценку за 5 минут
- Скорость: MVP за 1-2 недели (vs 4-8 недель у конкурентов)
- Прозрачное ценообразование: от 150K руб
- Мультимодельный подход: Claude + YandexGPT + GigaChat + open-source

## 3. Услуги и ценообразование

### Формула цены

Цена рассчитывается автоматически на основе реальных затрат:

```
ЦЕНА = (API_costs + Infra_costs + Ops_overhead + Taxes) × Markup

Где:
  API_costs     = токены Claude (input + output) + другие AI API
  Infra_costs   = хостинг, домен, SSL, БД, storage (pro rata на проект)
  Ops_overhead  = CAC (привлечение) + бухгалтерия + admin time (pro rata)
  Taxes         = 5% от прибыли (IT-аккредитация)
  Markup        = коэффициент наценки (см. ниже)
```

**Коэффициенты наценки по сложности:**

| Тип проекта | Markup | Почему |
|------------|--------|--------|
| Простой (бот, лендинг) | ×3.0 | Низкие API-расходы, высокая ценность для клиента |
| Средний (AI-ассистент, интеграция) | ×2.5 | Баланс затрат и ценности |
| Сложный (мультиагент, enterprise) | ×2.0 | Высокие API-расходы, но и большие чеки |
| Консалтинг | ×4.0 | Минимальные затраты, максимальная экспертиза |
| Поддержка (подписка) | ×2.0 | Recurring, предсказуемые расходы |

**Почему не 50% (×1.5), а ×2.0-4.0:**
- 50% наценка = маржа 33%. Это уровень продуктового ретейла, не IT-услуг
- Стандарт IT-аутсорса: маржа 50-70% (markup ×2.0-3.3)
- AI-студии мира работают с маржой 60-80%
- Основная ценность — не API-токены, а экспертиза, скорость, результат
- При ×1.5 один неоплаченный проект или сбой = убыток
- Рекомендация: ×2.5 как базовый множитель (маржа 60%)

### Калькуляция себестоимости по типам проектов

**Пример: AI чат-бот для Telegram (типовой проект)**

```
API-расходы:
  Сбор ТЗ (диалог):     ~$2  (Sonnet, ~50K токенов)
  Генерация ТЗ/КП:      ~$5  (Opus, ~30K токенов)
  Планирование:          ~$3  (Opus, ~20K токенов)
  Написание кода:        ~$15 (Sonnet, ~300K токенов, 5-8 сессий)
  Правки + итерации:     ~$10 (Sonnet, ~200K токенов)
  Итого API:             ~$35 = ~3 500 ₽

Инфра (pro rata на проект):
  VDS:                   500 ₽
  Supabase:              300 ₽
  Vercel:                200 ₽
  Итого инфра:           1 000 ₽

Операционные:
  CAC (доля маркетинга):  5 000 ₽
  Бухгалтерия (pro rata): 1 000 ₽
  Admin time:             2 000 ₽
  Итого ops:              8 000 ₽

Себестоимость:            12 500 ₽
Налог (5% от прибыли):   рассчитывается после
Markup ×3.0:              37 500 ₽
─────────────────────────
ЦЕНА КЛИЕНТУ:            ~50 000 ₽ (округл.)
```

Но! Рыночная цена AI чат-бота = от 150-200K. Значит:

```
Цена = MAX(себестоимость × markup, рыночный_минимум)
```

**AI выставляет цену автоматически**, учитывая оба фактора.

### Каталог услуг с реальной себестоимостью

| # | Услуга | Себестоимость | Markup | Мин. цена | Рыночная цена | Финальная цена |
|---|--------|-------------|--------|-----------|---------------|---------------|
| 1 | AI чат-бот (Telegram/сайт) | 12-25K | ×3.0 | 36-75K | 150-300K | **от 150K** |
| 2 | AI-ассистент (RAG) | 25-50K | ×2.5 | 62-125K | 300-500K | **от 300K** |
| 3 | AI-агент автоматизации | 40-80K | ×2.5 | 100-200K | 500K-1.5M | **от 500K** |
| 4 | Сайт + AI-функции | 15-35K | ×3.0 | 45-105K | 200-500K | **от 200K** |
| 5 | Интеграция AI (CRM/1С) | 20-40K | ×2.5 | 50-100K | 250-500K | **от 250K** |
| 6 | AI-консалтинг (аудит) | 3-8K | ×4.0 | 12-32K | 50-200K | **от 50K** |
| 7 | Подписка (поддержка) | 10-25K/мес | ×2.0 | 20-50K/мес | 30-150K/мес | **от 30K/мес** |

**Вывод:** реальная себестоимость проекта = 12-80K ₽. Основная стоимость для клиента — это ценность результата, а не API-токены. Маржа 75-90% при рыночном ценообразовании.

### Автоматический расчёт цены (AI)

Claude на этапе 3 (КП) считает цену по формуле:

```
1. Оценивает кол-во токенов на проект (по типу + фичам)
2. Считает API cost (Sonnet $3/1M input, $15/1M output; Opus $15/$75)
3. Добавляет infra + ops overhead
4. Применяет markup по типу проекта
5. Сравнивает с рыночным минимумом
6. Берёт MAX → финальная цена
7. Показывает клиенту разбивку: «Стоимость: X ₽, из них разработка Y, инфра Z»
```

Прайс-лист и коэффициенты настраиваются в админке (/admin/settings).

### Подключаемые API и интеграции

Claude Code для выполнения работы подключается к внешним сервисам через настраиваемые API.

**Исполнение (для каждого клиентского проекта):**

| Интеграция | Назначение | Как подключается |
|-----------|------------|-----------------|
| **GitHub API** | Создание репо, коммиты, ветки, PR, деплой через Actions | OAuth App / Fine-grained PAT |
| **VDS по SSH** | Деплой на сервер клиента, запуск сервисов, настройка nginx | SSH-ключ (генерируется per-project) |
| **Vercel API** | Preview-деплой, продакшн-деплой, env vars | Project-scoped token |
| **Railway API** | Деплой бэкенда, БД, воркеры | Project token (GraphQL) |
| **Claude API** | Генерация кода, ТЗ, КП, планирование | API key через @ai-sdk/anthropic |
| **YandexGPT API** | Русскоязычный контент, fallback | IAM-токен через Яндекс.Облако |
| **GigaChat API** | Fallback, задачи на русском | OAuth через Сбер |
| **Supabase API** | БД, auth, storage для клиентских проектов | Service key per project |

**Клиентские интеграции (настраиваются в ЛК / админке):**

| Интеграция | Назначение | Протокол |
|-----------|------------|----------|
| **Bitrix24** | CRM, сделки, контакты | REST API + webhooks |
| **1С** | Учёт, документы, товары | REST API / OData |
| **Telegram Bot API** | Уведомления, клиентские боты | Bot token |
| **VK API** | Боты для сообществ | Group token |
| **WhatsApp Business** | Чат-боты для бизнеса | Cloud API |
| **SMTP / Resend** | Email-рассылки | API key |
| **Яндекс.Метрика** | Аналитика | API |
| **Roistat / Calltouch** | Сквозная аналитика | API |
| **Stripe / ЮKassa** | Приём платежей | API key |
| **Cal.com / Calendly** | Запись на встречи | OAuth |

**Архитектура подключений:**

```
Админка (/admin/settings/integrations)
  ↓
Encrypted credentials store (Supabase Vault / env vars)
  ↓
n8n как middleware:
  - Webhook принимает событие от сайта
  - Выбирает нужные credentials
  - Выполняет action (deploy, push, API call)
  - Возвращает результат в ЛК + Telegram
  ↓
Claude Code использует credentials через tool calling:
  - deploy_to_github(repo, branch)
  - deploy_to_vds(host, path, commands)
  - deploy_to_vercel(project_id)
  - call_api(service, endpoint, payload)
```

**Безопасность:**
- Credentials хранятся в Supabase Vault (encrypted at rest)
- Доступ только через n8n / backend, никогда через фронтенд
- Per-project isolation: каждый проект видит только свои ключи
- Audit log: кто, когда, какой API вызвал
- SSH-ключи генерируются per-project, отзываются при закрытии

### Полный бизнес-процесс (воронка)

AI ведёт проект от первого контакта до сдачи. Человек (ты) — супервайзер на ключевых точках.

```
ЭТАП 1: ЗАЯВКА И СБОР ТЗ
─────────────────────────
Клиент заходит на сайт
  ↓
AI-чат виджет (Claude через Vercel AI SDK)
  ↓
Tool calling: сбор требований через диалог
  [тип проекта, фичи, дизайн, бюджет, сроки, контакты]
  ↓
Structured output → JSON (Zod-валидация)
  ↓
Генерация ТЗ (markdown → PDF)
  ↓
📩 Telegram: «Новая заявка — [тип] — [бюджет]»

ЭТАП 2: СОГЛАСОВАНИЕ ТЗ С КЛИЕНТОМ
────────────────────────────────────
AI отправляет ТЗ клиенту (email + личный кабинет)
  ↓
Клиент вносит правки через чат / комментарии в ЛК
  ↓
AI обновляет ТЗ → новая версия → повторное согласование
  ↓
Клиент подтверждает ТЗ ✔
  ↓
📩 Telegram: «ТЗ согласовано — [проект]»

ЭТАП 3: КОММЕРЧЕСКОЕ ПРЕДЛОЖЕНИЕ
─────────────────────────────────
AI генерирует КП на основе утверждённого ТЗ:
  - Стоимость (Claude + прайс-лист в контексте)
  - Сроки по этапам
  - Стек технологий
  - Условия оплаты
  ↓
КП → клиенту (PDF + веб-страница)
  ↓
Клиент принимает / торгуется (AI ведёт переговоры в рамках лимитов)
  ↓
КП утверждено ✔ → сделка в CRM
  ↓
📩 Telegram: «КП принято — [сумма] — старт работ»

ЭТАП 4: ПЛАН РАЗРАБОТКИ
────────────────────────
AI (Claude Code) декомпозирует ТЗ в план:
  - Разбивка на этапы и задачи
  - Оценка часов на каждую задачу
  - Зависимости между задачами
  - Milestone-ы для клиента
  ↓
План → клиенту на утверждение (ЛК + email)
  ↓
Клиент утверждает план ✔
  ↓
📩 Telegram: «План утверждён — [N задач] — старт [дата]»

ЭТАП 5: ВЫПОЛНЕНИЕ РАБОТЫ (Claude Code)
────────────────────────────────────────
Claude Code выставляет себе задачи из плана:
  ↓
Для каждой задачи:
  1. Создаёт ветку (git checkout -b task/название)
  2. Пишет код / контент / конфиг
  3. Тестирует (автотесты, линтер)
  4. Коммитит с описанием
  5. Обновляет прогресс в ЛК клиента
  ↓
📩 Telegram (прогресс): «Задача 3/8 выполнена — [название]»
  ↓
При блокере / неопределённости:
  → AI спрашивает клиента через чат в ЛК
  → или эскалирует тебе в Telegram
  ↓
📩 Telegram (milestone): «Этап 1 завершён — [ссылка на превью]»

ЭТАП 6: ПРЕЗЕНТАЦИЯ РЕЗУЛЬТАТА
───────────────────────────────
AI собирает демо:
  - Деплой на preview-URL (Vercel preview / staging)
  - Генерация отчёта: что сделано, скриншоты, метрики
  - Видео-превью (опционально: Puppeteer → скринкаст)
  ↓
Отправка клиенту:
  - Email: «Ваш проект готов к просмотру»
  - ЛК: отчёт + preview-ссылка + кнопка «Принять / Правки»
  ↓
📩 Telegram: «Результат отправлен клиенту — [проект]»

ЭТАП 7: ПРАВКИ
──────────────
Клиент оставляет замечания через ЛК / чат
  ↓
AI анализирует правки:
  - Входит в скоуп ТЗ? → выполняет
  - Выходит за скоуп? → предлагает доп. оценку
  ↓
Цикл: правки → демо → согласование
  (до 3 итераций в базовом пакете)
  ↓
📩 Telegram: «Правки внесены — итерация [N]»

ЭТАП 8: СДАЧА И ЗАКРЫТИЕ
─────────────────────────
Клиент принимает работу ✔
  ↓
AI выполняет:
  - Деплой на продакшн (если включён в скоуп)
  - Передача исходников (GitHub repo)
  - Генерация документации
  - Закрытие сделки в CRM
  - Отправка акта приёма-передачи
  ↓
Предложение подписки на поддержку (30K/мес+)
  ↓
📩 Telegram: «Проект [название] сдан ✅ — [сумма]»
```

### Система задач (self-tasking)

Claude Code управляет своей работой через внутреннюю систему задач:

```
Проект в CRM
  ↓
Парсинг утверждённого плана → массив задач
  ↓
Каждая задача:
  {
    id, title, description,
    status: pending | in_progress | review | done | blocked,
    priority: P0 | P1 | P2,
    estimated_hours, actual_hours,
    dependencies: [task_ids],
    branch: "task/xxx",
    commits: [sha],
    milestone: "этап-1"
  }
  ↓
Очередь выполнения (по приоритету + зависимостям)
  ↓
Прогресс → ЛК клиента (realtime через Supabase)
  ↓
Дашборд для тебя: все проекты, все задачи, статусы
```

### Почему маржа такая высокая

1. **API-расходы мизерные** — Claude стоит $7-110 на проект, а продаём за 150K-1M ₽
2. **Нет зарплат** — Claude Code = бесплатный «сотрудник» (платим только за API)
3. **IT-льготы** — налог 5% вместо 25%, взносы 15% вместо 30%
4. **Масштабируемость** — один Claude Code может вести 5-10 проектов параллельно
5. **Низкий CAC** — экспертный контент на vc.ru + SEO = почти бесплатные лиды

### Точка безубыточности

```
Постоянные расходы: ~12-16K/мес
Переменные на 1 проект: ~10-35K
Средний чек: 250K

Безубыточность = 1 проект за 2 месяца
Реалистично: выход в плюс с 1-го месяца при 1 проекте
```
